Test steps:

  1. Download the beta version(e.g 8.37.189.8) aar package of crosswalk(https://download.01.org/crosswalk/releases/crosswalk/android/maven2/org/xwalk/xwalk_core_library/)
  2. Open the console, and enter the usecase-embedding-android-tests directory
  3. Run 'mvn install:install-file -DgroupId=com.xwalk -DartifactId=xwalk_core_library -Dversion=9.38.206.0 -DgeneratePom=true -Dpackaging=aar -Dfile=xwalk_core_library.aar -DlocalRepositoryPath=/path/to/usecase-embedding-android-tests/libs' to install the downloaded aar package
  4. Set the url of Dependencies in build.gradle to 'libs'
  5. Run "./gradlew assemblex86" command on the console

Expected Output:

  1. The downloaded aar package could be installed successfully on the console
  2. The libs directory of usecase-embedding-android-tests is not empty
  3. "BUILD SUCCESSFUL" is shown on the console info
  4. Two apk files(usecase-embedding-android-tests-x86-debug-unaligned.apk and usecase-embedding-android-tests-x86-release-unsigned.apk) are built in usecase-embedding-android-tests/build/apk directory
  5. The usecase-embedding-android-tests-x86-debug-unaligned.apk could be installed, uninstalled, launched and performed correctly on IA phone
