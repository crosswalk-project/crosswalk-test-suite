Test that a separate 64-bit Crosswalk APK support in Google Play store

Pre-condition:

  1. A 64bit IA device.
  2. Uninstall pre-installed Crosswalk runtime library.
  3. Google Play store or other PRC store are installed.

Test steps:

Device ArchitectureApp ModeExpected Crosswalk installed via Google Play StoreExpected Crosswalk installed via PRC Store
IA 64bit32bit-ARM-embeddedorg.xwalk.coreorg.xwalk.core.ia
IA 64bit32bit-ARM-embedded + Sharedorg.xwalk.core + org.xwalk.core64org.xwalk.core.ia + org.xwalk.core64.ia
  1. Find(or Build) a Crosswalk 32 bit ARM apk in embedded mode, install it to 64 bit IA device, and launch it.
  2. Get Crosswalk via Google Play store or other PRC store.
  3. Launch the embedded app again.
  4. Check installed Crosswalk runtime library by command line.
  5. Find(or Build) a Crosswalk shared apk, install it to the device, and launch it.
  6. Get Crosswalk via Google Play store or other PRC store.
  7. Launch the shared app again.
  8. Check installed Crosswalk runtime library by command line.
  9. Uninstall the installed Crosswalk runtime library.

Reference command lines:

  1. Building shared mode apk:
    $ crosswalk-pkg --crosswalk=[crosswalk-version] --platform=android --android=[shared|embedded] --targets=[arm|x86] /path/to/testapps/demo
  2. Finding installed Crosswalk runtime library:
    $ adb shell dumpsys|grep org.xwalk.core
