Launch sub app "iap"

Verifies query product details and buy gas product successfully.

Pre-condition:

  1. GMS
    IAP is provided by GMS, which means the device must have Google Play installed, and be able to access Google Play.
    Here is a wiki of setup steps for GMS on ZTE Geek for your reference: http://powerbuilder.sh.intel.com/projects/wrtonandroid/wiki/xwalk/geek_install_gms
  2. Google Play developer account
    IAP must have products for user to buy, these products can only be managed in Google Play developer console by developer. The account on the device which you are using to test your IAP app should be the test account listed in your developer console, for more information please see Testing an APK with Google In-App Billing API
  3. Google Wallet merchant account
    User must have Google Wallet merchant account to use IAP service on Google Play. In our scenario, "user" is the user account on test device - ZTE Geek. Note this account must be different with the developer account in above, use for purchase.
  4. No GFW
    IAP is not provided in China mainland, we must find a way to go over GFW to test IAP functionalities.
