Preconditions

  1. Require npm
  2. Require plugman
    $ sudo npm install ­-g plugman
    Please make sure the version of plugin is >= 0.22.17, Use "plugman ­-v" common to check it

Test Steps

  1. Download crosswalk-cordova--arm.zip and unzip it
  2. Create Cordova example app, e.g. “hello”:
    3. $ cd crosswalk-cordova-
    $ ./bin/create hello com.example.hello hello
  3. Setup Cordova example app:
    $ cd hello
    $ sed -i "s/<\/widget>/    <preference name=\"Fullscreen\" value=\"true\" \/>\\n<\/widget>/g" res/xml/config.xml
    $ sed -i "s/<\/widget>/    <preference name=\"disableImmersive\" value=\"true\" \/>\\n<\/widget>/g" res/xml/config.xml
    $ plugman install --platform android --project . --plugin org.apache.cordova.statusbar@0.1.10
  4. Copy the 'usecase-cordova-android-tests/res/StatusBar/index.html' file to hello/assets/www directory
  5. Build Cordova example app:
    $ ./cordova/build
  6. Check the apk file generated in hello/bin directory
  7. Install, Launch, Run, Click the buttons and Exit the app

Purpose:

Verify the Cordova StatusBar plugin can work when app in fullscreen mode

Expected Results:

  1. Create test suite(hello) successfully
  2. Build the apk successfully
  3. The app can be installed successfully
  4. The app can launch successfully
  5. The app can run successfully
  6. The app show the StatusBar after click the "Status Bar Show" button
  7. The app hide the StatusBar after click the "Status Bar Hide" button
  8. The app can exit successfully
