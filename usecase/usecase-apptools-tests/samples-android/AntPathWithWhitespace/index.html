Precondition

  1. Please refer to README file

Test Steps

  1. If org.xwalk.test and lastest stable version crosswalk exist in usecase-apptools-tests/tools/ directory, remove it
    Then run below commands under usecase-apptools-tests/tools/ directory:
    New org.test.foo folder
    $ cd org.test.foo
    On Linux or OS X host:
        Install apache ant and whitespace in path to apache ant, e.g. in Downloads/apache\ ant/...
        Shared Mode:
            $ crosswalk-pkg --manifest="org.test.foo" --android=shared --targets="armeabi-v7a/arm64-v8a x86/x86_64" ./
        Embedded Mode:
            $ crosswalk-pkg --manifest="org.test.foo" --targets="armeabi-v7a/arm64-v8a x86/x86_64" ./
        Lite Mode:
            $ crosswalk-pkg --manifest="org.test.foo" --android=lite --targets="armeabi-v7a/arm64-v8a x86/x86_64" ./
    On Windows host:
        Install apache ant and whitespace in path to apache ant, e.g. in C:\Program Files\...
        Shared Mode:
            $ node %crosswalk-pkg% --manifest="org.test.foo" --android=shared --targets="armeabi-v7a/arm64-v8a x86/x86_64" .\
        Embedded Mode:
            $ node %crosswalk-pkg% --manifest="org.test.foo" --targets="armeabi-v7a/arm64-v8a x86/x86_64" .\
        Lite Mode:
            $ node %crosswalk-pkg% --manifest="org.test.foo" --android=lite --targets="armeabi-v7a/arm64-v8a x86/x86_64" .\
    Launch the app on android device

Purpose:

Validate if app can be built when whitespace in path to apache ant

Expected Result:

  1. Success to build and launch the app
